# Processed Skin Cancer Dataset Documentation

This document describes the structure, origin, and creation pipeline for the unified, training-ready skin cancer dataset located in the `processed/` directory.

## Dataset Overview

The dataset is a unified collection of dermoscopic images prepared for multi-class skin lesion classification. It consolidates samples from two primary Kaggle/ISIC sources:
1. **HAM10000** (`kmader/skin-cancer-mnist-ham10000`): Contains 10,015 images representing 7 distinct diagnostic categories.
2. **ISIC 2018 Challenge Task 3**: Sourced for validation ground truth matching the HAM10000 image series.

The combined dataset contains **10,015 unique images** classified into 7 disease categories.

## Creation Pipeline (`fix_dataset.py`)

The processed dataset was generated by running the standalone script `fix_dataset.py` which performs the following sequential steps:

1. **Metadata Loading & Image Mapping**:
   - Loads the patient metadata from HAM10000 and aligns it with the ISIC 2018 Task 3 Ground Truth labels.
   - Maps each image ID to its path in the localized dataset folders.
2. **Deduplication**:
   - Resolves overlap between the HAM10000 and ISIC 2018 Task 3 training sets (which share identical image IDs) by dropping duplicate paths, retaining exactly one copy per unique image.
3. **Image Integrity Verification**:
   - Verifies the byte integrity of all mapped images using `Pillow`'s `verify()`. Corrupted or unreadable files are automatically excluded.
4. **Stratified Split**:
   - Splits the validated images into **Train (80%)**, **Validation (10%)**, and **Test (10%)** sets.
   - The split is stratified by lesion classification labels to preserve class distributions across all subsets.
5. **Class Balancing via Oversampling**:
   - Skin cancer datasets are heavily imbalanced (e.g., `nv` dominates the distribution).
   - The training set is balanced by upsampling all minority classes (with replacement) to match the count of the majority class (`nv`, 5,364 samples).
   - Validation and test splits are left unbalanced to represent real-world clinical distributions.
6. **Export**:
   - Copies images into class-specific folders (`train/`, `val/`, `test/`). 
   - Appends duplicate suffixes (e.g., `_dup2`) for oversampled training images to avoid filename collisions.
